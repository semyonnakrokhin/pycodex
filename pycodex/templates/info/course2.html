Добро пожаловать на курс "Backend на FastAPI и введение в архитектуру web-приложений"!


Этот курс предназначен для всех, кто желает освоить разработку backend-части веб-приложений с использованием FastAPI и получить введение в архитектурные концепции веб-приложений.

Цель программы обучения:

  1. Познакомить вас с базовыми принципами разработки backend-части веб-приложений.
  2. Освоить использование FastAPI - современного и эффективного фреймворка для разработки веб-приложений на Python.
  3. Разработать навыки работы с API, обработкой запросов и ответов.
  4. Изучить основные архитектурные принципы и шаблоны веб-приложений.
  5. Предоставить практический опыт в создании эффективного и масштабируемого backend-кода.
  6. Ознакомить с инструментами тестирования и отладки веб-приложений.
  7. Обучить работе с базами данных, взаимодействию с ними через ORM (Object-Relational Mapping).
  8. Представить принципы безопасности веб-приложений и методы их обеспечения.

Программа обучения включает в себя модули, посвященные как базовым концепциям разработки backend, так и практическим навыкам работы с FastAPI и архитектурными принципами веб-приложений. Каждый модуль содержит теоретические материалы, примеры кода и практические упражнения, чтобы помочь вам закрепить изученный материал.

Мы рекомендуем следовать программе обучения поэтапно и активно применять полученные знания на практике. По завершении курса вы будете готовы не только создавать backend-часть веб-приложений с использованием FastAPI, но и понимать основные принципы архитектуры веб-приложений, что поможет вам в дальнейшем развитии в этой области.

Желаем вам удачи в изучении "Backend на FastAPI и введение в архитектуру web-приложений"!

Критерии отбора

1. Успешное прохождение интервью по темам базового Python и ООП

2. Успешное решение задачи следующего уровня в течение 1 дня:

Необходимо объявить класс с именем TicTacToe (крестики-нолики) для управления игровым процессом. Объекты этого класса будут создаваться командой: 

game = TicTacToe()

В каждом объекте этого класса должен быть публичный атрибут: pole - двумерный кортеж, размером 3x3.

Каждый элемент кортежа pole является объектом класса Cell: 

cell = Cell()

В объектах этого класса должно автоматически формироваться локальное свойство: value - текущее значение в ячейке: 0 - клетка свободна; 1 - стоит крестик; 2 - стоит нолик.

Также с объектами класса Cell должна выполняться функция: bool(cell) - возвращает True, если клетка свободна (value = 0) и False - в противном случае.

К каждой клетке игрового поля должен быть доступ через операторы: 

res = game[i, j] # получение значения из клетки с индексами i, j
game[i, j] = value # запись нового значения в клетку с индексами i, j

Если индексы указаны неверно (не целые числа или числа, выходящие за диапазон [0; 2]), то следует генерировать исключение командой 

raise IndexError('некорректно указанные индексы')

Чтобы в программе не оперировать величинами: 0 - свободная клетка; 1 - крестики и 2 - нолики, в классе TicTacToe должны быть три публичных атрибута (атрибуты класса): 

FREE_CELL = 0      # свободная клетка
HUMAN_X = 1        # крестик (игрок - человек)
COMPUTER_O = 2     # нолик (игрок - компьютер)

В самом классе TicTacToe должны быть объявлены следующие методы (как минимум):

init() - инициализация игры (очистка игрового поля, возможно, еще какие-либо действия);
show() - отображение текущего состояния игрового поля (как именно - на свое усмотрение);
human_go() - реализация хода игрока (запрашивает координаты свободной клетки и ставит туда крестик);
computer_go() - реализация хода компьютера (ставит случайным образом нолик в свободную клетку).

Также в классе TicTacToe должны быть следующие объекты-свойства (property):

is_human_win - возвращает True, если победил человек, иначе - False;
is_computer_win - возвращает True, если победил компьютер, иначе - False;
is_draw - возвращает True, если ничья, иначе - False.

Наконец, с объектами класса TicTacToe должна выполняться функция:

bool(game) - возвращает True, если игра не окончена (никто не победил и есть свободные клетки) и False - в противном случае. 

game = TicTacToe()
game.init()
step_game = 0
while game:
game.show()

if step_game % 2 == 0:
    game.human_go()
else:
    game.computer_go()

step_game += 1

game.show()

if game.is_human_win:
print("Поздравляем! Вы победили!")
elif game.is_computer_win:
print("Все получится, со временем")
else:
print("Ничья.")

Вам в программе необходимо объявить только два класса: TicTacToe и Cell так, чтобы с их помощью можно было бы сыграть в "Крестики-нолики" между человеком и компьютером.
